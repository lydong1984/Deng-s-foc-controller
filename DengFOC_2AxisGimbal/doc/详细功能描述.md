# DengFOC 双轴万向节 FOC 控制系统 - 详细功能描述

## 00. 项目概述

DengFOC 是一个基于 ESP32 微控制器的双轴万向节 FOC（场定向控制）驱动系统。该系统通过 FOC 控制算法实现对两个直流无刷电机的精准控制，结合 IMU 传感器和磁编码器反馈，实现高精度的角度/力矩控制。

**目标应用**: 云台、云台稳定器、航拍吊舱等机械稳定系统

## 01. 快速使用与标定

本节根据附件中的《DengFOC 自稳二轴云台用户手册》截图补充，用于把实物套件、板卡版本、标定烧录和实机上电流程纳入功能描述，便于从“代码功能”映射到“用户可执行操作”。

### 2. 套餐与安装识别

附件中的套件图显示，结构件包含 2208 电机 2 个、双口编码器、单口编码器、陀螺仪、`M3*6` 螺丝、`M2.5*6` 螺丝、`M2*6` 螺丝和磁铁 2 个等物料。功能描述中需要区分“结构件套件”和“驱动板套件”：

- **结构件散件套装**: 需要按装配资料完成机械结构、编码器、磁铁和陀螺仪安装。
- **结构件整机套装**: 云台机构已装好，可直接进入电气连接、标定和运行准备。
- **驱动板依赖**: 结构件套件不等同于 DengFOC 驱动板套件，运行系统仍需 DengFOC V3P 或 DengFOC V4 驱动板。

### 3. 标定、快速启动与运行状态

系统使用 `Calibration_Flag` 控制 EEPROM 标定写入流程：

| 阶段 | `Calibration_Flag` | 用户动作 | 系统行为 |
|------|--------------------|----------|----------|
| 标定自检 | `1` | 断开 12V 总电源，仅保留 USB Type-C 烧录；烧录后打开串口工具并连接 12V 总电源 | 执行 M0/M1 电机对齐，记录零电角度和编码器方向，写入 EEPROM |
| 快速启动 | `0` | EEPROM 写入成功后改回 `0` 并再次烧录 | 从 EEPROM 读取零电角度和方向，不再重复写入标定数据 |
| 实机运行 | `0` | 上电前调整云台电机到可自由运动、避开机械卡位的位置 | 初始化 BMI160、读取编码器标定参数，进入 `runFOC()` 实时稳定控制 |

附件强调的上电顺序是：**烧录前断开总电源，只连接 USB Type-C；标定或运行观察时再连接 12V 总电源**。这样可以避免烧录过程受电机驱动供电影响，也能降低电机意外动作风险。

### 4. PlatformIO 调试路径

PlatformIO 路径面向 VS Code + PlatformIO IDE 工程：

7. 点击 PlatformIO 编译按钮，再点击烧录按钮。
8. 打开 VOFA+，选择前面记录的 COM 端口。
9. 调整云台机构到可自由运动的位置，连接 12V 总电源，观察自检和 EEPROM 写入结果。
10. 写入成功后设置 `#define Calibration_Flag 0`，再次烧录，进入快速启动和实机运行流程。

### 5. Arduino 调试路径

Arduino 路径面向 Arduino IDE 工程：

1. 使用 Arduino IDE 打开 `DengFOC_2AxisGimbal` 程序。
2. 通过 USB Type-C 连接驱动板和电脑，在设备管理器中确认 `CH340 COM` 端口。
3. 在工具菜单中选择 ESP32 开发板、对应板卡型号和端口。
4. 安装或确认 BMI160 依赖库。
5. 根据驱动板设置 `#define DengFOC`：V3P 为 `0`，V4 为 `1`。
6. 首次标定时设置 `#define Calibration_Flag 1`。
7. 断开 12V 总电源，仅通过 USB Type-C 编译和烧录。
8. 打开 Arduino 串口监视器。
9. 调整云台机构到可自由运动的位置，连接 12V 总电源，执行自检。
10. EEPROM 写入成功后改为 `#define Calibration_Flag 0` 并再次烧录。

## 02. 编码器角度功能详细描述

AS5600 编码器是 DengFOC 系统中的关键组件，用于实时测量电机转子的角度位置。该编码器采用磁性传感技术，通过检测磁铁的位置来输出精确的角度值，为 FOC（场定向控制）算法提供必要的角度反馈。

### 1. 编码器工作原理

- **传感器类型**: 磁性角度传感器，使用霍尔效应原理检测磁场方向，无需物理接触。
- **测量范围**: 0° 到 360°，支持连续旋转测量，适用于电机全角度控制。
- **通信接口**: I2C 接口，与 ESP32 微控制器通信，数据传输稳定可靠。
- **分辨率**: 12位数字输出（4096 个位置），角度分辨率约为 0.0879°，满足高精度定位需求。
- **输出数据**: 提供原始角度值和经过滤波的角度值，支持实时读取。

### 2. 在 FOC 控制中的作用

- **角度反馈**: 提供电机转子的实时角度位置，用于 FOC 算法中的 Clarke 变换和 Park 变换，实现精确的磁场定向控制。
- **闭环控制**: 结合 IMU（惯性测量单元）数据，实现双轴云台的高精度角度和力矩控制，确保云台稳定。
- **速度控制**: 通过角度变化率计算电机转速，为速度环控制提供反馈。
- **位置控制**: 支持绝对位置控制，电机可精确定位到指定角度。

### 3. 标定过程

- **标定标志**: 当 `Calibration_Flag = 1` 时，系统执行电机对齐过程，自动检测并记录零电角度和编码器旋转方向。
- **EEPROM 存储**: 标定数据（零电角度和方向）存储在 ESP32 的 EEPROM 中，避免每次上电重复标定，提高系统启动效率。
- **方向检测**: 系统自动检测编码器旋转方向（正向或反向），确保角度反馈与电机实际旋转一致。
- **自检机制**: 标定过程中系统会验证编码器连接和数据有效性，防止标定失败导致的控制问题。

#### 3.1 标定流程

1. **准备阶段**
   - 确认机械结构、编码器、磁铁已正确安装。
   - 断开 12V 总电源，仅通过 USB Type-C 与电脑连接。
   - 打开串口监视器或 VOFA+，设置正确的 COM 端口和波特率（115200）。

2. **配置与烧录**
   - 将源码中的 `#define Calibration_Flag 1` 设置为标定模式。
   - 根据实际主板型号设置 `#define DengFOC`（V3P 为 `0`，V4 为 `1`）。
   - 编译并烧录程序到 ESP32。

3. **连接总电源**
   - 在程序烧录完成后，连接 12V 总电源。
   - 保持云台结构处于可自由运动状态，避免机械卡位。

4. **执行标定**
   - 系统自动启动电机对齐流程，读取 AS5600 编码器角度。
   - 记录 M0/M1 的零电角度和编码器方向，并将标定数据写入 EEPROM。
   - 串口输出应显示标定成功、零电角度值和方向检测结果。

5. **验证与保存**
   - 如果标定失败，检查编码器连接、磁铁位置和 I2C 总线状态。
   - 确认 EEPROM 写入成功后，断开 12V 总电源。
   - 将 `#define Calibration_Flag` 改回 `0`，重新编译并烧录程序。

6. **快速启动验证**
   - 重新连接 12V 总电源并启动系统。
   - 系统应从 EEPROM 读取标定数据，而不是重新执行标定流程。
   - 观察系统进入 `runFOC()` 并完成正常自稳控制。

### 4. 精度与可靠性

- **精度**: 典型精度为 ±0.5°，在云台稳定器等应用中可实现亚度级控制精度。
- **抗干扰能力**: 磁性传感技术不易受电磁干扰影响，适合电机驱动环境。
- **温度稳定性**: 在 -40°C 到 +125°C 工作温度范围内保持稳定性能。
- **耐用性**: 无机械磨损，寿命长，适合长期运行的云台系统。

### 5. 集成与配置

- **硬件连接**: 编码器通过 I2C 总线连接到 ESP32，支持多编码器级联（M0 和 M1 电机各一个）。
- **软件接口**: 通过 DengFOC 库的 AS5600 类实现角度读取，接口简单易用。
- **配置选项**: 支持角度偏移校正和滤波参数调整，以适应不同应用需求。

### 6. AS5600 功能描述与框图

AS5600 是系统中的磁性角度编码器模块，负责将电机转子的磁场方向转换为数字角度信息，并将结果通过 I2C 发送给 ESP32。它的主要功能包括:

- **角度测量**: 提供 0°~360° 的绝对角度值，满足双轴电机的位置反馈需求。
- **速度推算**: 通过连续角度采样计算转速，用于速度环控制。
- **方向识别**: 支持正反转方向检测，避免编码器读数与电机实际转向不一致。
- **标定与零点保存**: 与系统标定流程配合，记录零电角度和偏移量，保证每次启动后位置反馈一致。
- **异常检测**: 支持检测 I2C 通信错误和数据无效状态，防止失控风险。

## 04. 核心模块详解

### 1. 传感器模块

#### 1.1 IMU 传感器 (BMI160)
- **功能**: 获取陀螺仪和加速度计数据
- **数据**: 
  - 加速度: ax, ay, az (原始值) → accX, accY, accZ (g单位)
  - 角速度: gx, gy, gz (原始值) → gyroX, gyroY, gyroZ (dps单位)
- **校准**: 启动时自动陀螺仪校准，消除零偏 (gyroXoffset, gyroYoffset, gyroZoffset)
- **应用**: 
  - 角度融合计算
  - 稳定性检测（判断电机是否静止）

#### 1.2 磁编码器 (AS5600)
- **数量**: 2个 (M0和M1各一个)
- **功能**: 
  - 精准角度反馈
  - 速度计算 (通过角度微分)
  - 完整圈数计数 (full_rotations)
- **精度**: 12位分辨率 (0.0879°/LSB)
- **通信**: 两路 I2C (通道0: M0, 通道1: M1)

#### 1.3 电流检测 (InlineCurrent)
- **功能**: A/B相电流在线检测
- **应用**: 电流闭环控制、过流保护

## 05. 扭矩控制与速度控制的详细对比

### 1. 控制原理的根本区别

#### 1.1 直流无刷电机的基本方程

直流无刷电机的数学模型可以用以下方程组表示：

**电压方程**:
$$U_q = R \cdot I_q + L \frac{dI_q}{dt} + K_e \cdot \omega$$

其中：
- $U_q$: q轴电压（控制电压）
- $R$: 电机绕组电阻
- $I_q$: q轴电流（控制电流）
- $L$: 电机绕组电感
- $K_e$: 电机反电动势常数
- $\omega$: 电机角速度

**力矩方程** (Newton第二定律):
$$T = K_t \cdot I_q$$

其中 $K_t$ 是力矩常数（直流无刷电机中 $K_t = K_e$）

**机械动力学方程**:
$$T_{motor} - T_{load} - B \cdot \omega = J \frac{d\omega}{dt}$$

其中：
- $T_{motor}$: 电机产生的力矩
- $T_{load}$: 负载力矩
- $B$: 阻尼系数
- $J$: 转动惯量

#### 1.2 扭矩控制原理

```mermaid
graph LR
    A["目标扭矩<br/>T_target"] -->|扭矩转换| B["目标电流<br/>Iq_target = T_target / Kt"]
    B -->|电流控制<br/>PID| C["q轴电压<br/>Uq"]
    C -->|Park反变换| D["3相电压<br/>Ua, Ub, Uc"]
    D -->|SVPWM| E["PWM占空比"]
    E -->|驱动| F["直流无刷电机"]
    F -->|反馈电流| B
```

**扭矩控制的控制目标**:
$$T_{target} = f(t)$$

直接指定目标扭矩，通过电流环PID控制使实际电流跟踪目标电流：

$$I_q^* = \frac{T_{target}}{K_t}$$

**控制流程**:
```
1. 获得目标扭矩 T_target
2. 计算目标电流: I_q_target = T_target / K_t
3. 获得实际电流反馈: I_q_actual
4. 电流误差: e_I = I_q_target - I_q_actual
5. 电流环PID: U_q = PID(e_I)
6. 输出PWM驱动电机
```

**数学表示**:
$$U_q(t) = K_p \cdot e_I(t) + K_i \int_0^t e_I(\tau)d\tau + K_d \frac{de_I}{dt}$$

其中 $(K_p, K_i, K_d)$ 是电流环的PID系数。

#### 1.3 速度控制原理

```mermaid
graph LR
    A["目标速度<br/>ω_target"] -->|速度误差| B["e_ω = ω_target - ω_actual"]
    B -->|速度环PID| C["目标扭矩<br/>T_target"]
    C -->|扭矩转换| D["目标电流<br/>Iq_target = T_target / Kt"]
    D -->|电流环PID| E["q轴电压<br/>Uq"]
    E -->|Park反变换| F["3相电压<br/>Ua, Ub, Uc"]
    F -->|SVPWM| G["PWM占空比"]
    G -->|驱动| H["直流无刷电机"]
    H -->|反馈速度| B
```

**速度控制的控制目标**:
$$\omega(t) = \omega_{target}(t)$$

通过速度环PID将速度误差转化为目标扭矩，再通过电流环PID实现扭矩控制：

**速度环PID**:
$$T_{target}(t) = K_{p,\omega} \cdot e_{\omega}(t) + K_{i,\omega} \int_0^t e_{\omega}(\tau)d\tau + K_{d,\omega} \frac{de_{\omega}}{dt}$$

其中：
- $e_{\omega} = \omega_{target} - \omega_{actual}$
- $(K_{p,\omega}, K_{i,\omega}, K_{d,\omega})$ 是速度环的PID系数

**电流环PID** (嵌套):
$$U_q(t) = K_{p,I} \cdot e_I(t) + K_{i,I} \int_0^t e_I(\tau)d\tau + K_{d,I} \frac{de_I}{dt}$$

**控制流程**:
```
1. 获得目标速度 ω_target
2. 获得实际速度反馈: ω_actual (从编码器计算)
3. 速度误差: e_ω = ω_target - ω_actual
4. 速度环PID: T_target = PID_vel(e_ω)
5. 电流目标: I_q_target = T_target / K_t
6. 电流环PID: U_q = PID_cur(e_I)
7. 输出PWM驱动电机
```

### 3. 动态响应对比

#### 3.1 单位阶跃响应

```mermaid
graph LR
    subgraph "扭矩控制"
        A["T_target阶跃变化"] -->|快速响应| B["电流立即跟踪"]
        B -->|电机加速| C["速度线性增长"]
    end
    
    subgraph "速度控制"
        D["ω_target阶跃变化"] -->|速度环调节| E["T_target缓慢增长"]
        E -->|电流缓慢增长| F["速度指数增长"]
    end
```

**时间域对比**:

扭矩控制响应方程（假设忽略电感和反EMF的高阶项）：
$$\omega(t) = \frac{T_{target}}{B+J\frac{d}{dt}}$$

在阶跃输入下：
$$\omega(t) = \frac{T_{target} \cdot J}{B \cdot J} (1 - e^{-\frac{B}{J}t}) = \frac{T_{target}}{B}(1 - e^{-\frac{B}{J}t})$$

速度控制响应方程（设速度环和电流环时常数分别为 $\tau_1$ 和 $\tau_2$）：
$$\omega(t) = \omega_{target}(1 - e^{-t/(\tau_1 + \tau_2)})$$

#### 3.2 抗负载能力

**扭矩控制**: 给定固定扭矩 $T$，负载增加时：
$$\omega_{steady} = \frac{T - T_{load}}{B}$$

负载增加直接导致速度下降，系统无法补偿。

**速度控制**: 给定目标速度 $\omega_{target}$，负载增加时：
$$T = T_{load} + B \cdot \omega_{target} \text{ (速度环自动调节)}$$

系统自动增加扭矩以维持目标速度。

### 4. 参数优化与调试

#### 4.1 扭矩控制的PID调参

仅需调试电流环PID参数：

```cpp
// 电流环参数(DengFOC.cpp)
PIDController current_loop_M0 = PIDController{
    .P = 1.2,      // 比例增益 - 控制响应速度
    .I = 0.0,      // 积分增益 - 消除稳态误差
    .D = 0.0,      // 微分增益 - 阻尼震荡
    .ramp = 100000,// 输出斜率限制
    .limit = 12.6  // 最大输出限制(V)
};
```

**调参步骤**:
1. 从 P=1.0 开始，逐步增加 P 值直到出现轻微震荡
2. 根据需要增加 I 值消除稳态误差
3. 根据超调情况调整 D 值

**优化目标**:
- 快速跟踪目标电流
- 最小化超调 (< 10%)
- 消除稳态误差 (< 1%)

#### 4.2 速度控制的PID调参

需要调试两层PID参数，通常采用"外环缓、内环快"的原则：

```cpp
// 速度环参数(main.cpp中的Y_loop)
PIDController Y_loop = PIDController{
    .P = 0.045,    // 速度环比例增益 - 较小
    .I = 0.002,    // 速度环积分增益
    .D = 0.0001,   // 速度环微分增益
    .ramp = 100000,// 输出(扭矩)斜率限制
    .limit = 6     // 最大目标扭矩限制
};

// 电流环参数 - 响应更快
PIDController current_loop = PIDController{
    .P = 1.2,      // 电流环比例增益 - 较大
    .I = 0.0,
    .D = 0.0,
    .limit = 12.6
};
```

**调参步骤**:
1. 先固定外环参数，调试内环(电流环)使其快速稳定
2. 再调整外环(速度环)参数，响应时间通常为内环的10~100倍
3. 采用Ziegler-Nichols法则或试错法

**参数关系**:
$$\tau_{outer} >> \tau_{inner}$$
$$K_{p,outer} << K_{p,inner}$$

| **应用限制** | 需要已知目标状态 | 无目标速度指示时难以应用 |

#### 5.3 具体应用场景深度分析

##### 5.3.1 云台稳定（推荐：扭矩控制）

**应用描述**:
- 航拍云台、手持稳定器、摄影机云台等
- 需要快速响应IMU传感器的倾斜信息
- 通过精确扭矩控制抵消倾斜和晃动

**技术方案**:
```
IMU检测倾斜角度 Δθ
    ↓
计算误差角速度目标值
    ↓
转换为目标扭矩 T = Kp × Δθ + Ki × ∫Δθdt
    ↓
使用扭矩控制立即执行
    ↓
快速消除倾斜 (< 100ms)
```

**核心优势**:
- ✅ 毫秒级响应 (< 5ms) - 人眼无法感知延迟
- ✅ 无需编码器反馈 - 仅需IMU
- ✅ 低功耗 - 精确控制避免过度调节
- ✅ 自然感觉 - 跟随运动更平顺

**参数建议**:
```cpp
// 扭矩控制参数 - 云台稳定
float Kp = 0.045;  // 快速响应
float Ki = 0.002;  // 消除稳态偏差
float Kd = 0.0001; // 阻尼震荡
float torque_limit = 6.0;  // 扭矩限制
```

**典型性能指标**:
- 响应时间: 2-5ms
- 稳定精度: ±0.5°
- 功耗: 2-5W
- 最大倾斜角速度: 500°/s

##### 5.3.2 固定速度运行（推荐：速度控制）

**应用描述**:
- 传送带驱动、转台、恒速电机等
- 负载可能变化（货物增减、摩擦变化等）
- 需要保持恒定转速

**技术方案**:
```
用户设定目标转速 ω_target
    ↓
读取编码器实际转速 ω_actual
    ↓
速度环PID计算目标扭矩
    ↓
电流环执行扭矩控制
    ↓
负载变化时自动调节扭矩维持转速
```

**核心优势**:
- ✅ 负载变化 ±50% 仍保持稳定转速
- ✅ 无需频繁调参 - 参数鲁棒性强
- ✅ 长期稳定运行 - 适合工业应用
- ✅ 自动扭矩补偿 - 对用户透明

**参数建议**:
```cpp

float speed_Kd = 0.01;

```

**典型性能指标**:
- 转速精度: ±2%
- 响应时间: 100-500ms
- 负载承受范围: ±50%
- 稳定性: 可运行 > 10小时

##### 5.3.3 精确定位（推荐：速度角度控制）

**应用描述**:
- 相机自动对焦、伺服定位、机械臂关节等

- 同时要求加速度平稳

**技术方案**:

- 同时需要平稳运动 (速度控制优势)
- 工作环境多变

**采用的混合策略**:
```cpp

    GetAngle();           // 获取IMU+编码器角度
    
    if (系统静止 && IMU数据可信) {
        // 阶段1: 高速反应 - 扭矩控制
        float torque = Kp × angle_error + Kd × angle_velocity;
        setTorque(torque);        // 扭矩控制
        
    } else if (系统运动 && 需要平稳) {
        // 阶段2: 平稳运动 - 速度控制
        speed_error = speed_target - speed_current;
        torque = speed_loop.update(speed_error);
        setTorque(torque);        // 速度环输出的扭矩
        
    } else {
        // 阶段3: 角度闭环 - 级联控制
        angle_error = angle_target - angle_current;
        speed_target = angle_loop.update(angle_error);
        torque = speed_loop.update(speed_target - speed_current);
        setTorque(torque);
    }
}
```

**性能指标**:
- 稳定时间: < 100ms
- 稳定精度: ±0.5°
- 平稳性: 良好
- 适应性: 多工况支持

### 6. 数学推导汇总

#### 6.1 频域分析（Laplace变换）

**扭矩控制传递函数**:
$$G_{torque}(s) = \frac{\omega(s)}{T_{target}(s)} = \frac{1}{Js + B}$$

**速度控制传递函数** (带速度环和电流环):
$$G_{speed}(s) = \frac{\omega(s)}{\omega_{target}(s)} = \frac{G_{vel}(s) \cdot G_{cur}(s)}{1 + G_{vel}(s) \cdot G_{cur}(s)}$$

其中：
- $G_{vel}(s)$: 速度环PID
- $G_{cur}(s)$: 电流环PID

#### 6.2 稳态误差分析

**扭矩控制的稳态速度**:
$$\omega_{ss} = \frac{T - T_{load}}{B}$$

当负载改变时：
$$\Delta \omega_{ss} = \frac{\Delta T_{load}}{B}$$

**速度控制的稳态速度**:
$$\omega_{ss} = \omega_{target} \text{ (零误差)}$$

即使负载变化，系统仍维持目标速度。

#### 6.3 能量考量

扭矩控制下的功率消耗：
$$P = T \cdot \omega = T \cdot \frac{T - T_{load}}{B}$$

速度控制下的功率消耗（负载补偿）：
$$P = T \cdot \omega_{target} = (T_{load} + B\omega_{target}) \cdot \omega_{target}$$

### 7. 实现建议

#### 7.1 选择扭矩控制的条件
- ✅ 系统具有IMU等外部反馈（已知目标状态）
- ✅ 需要毫秒级快速响应
- ✅ 控制对象负载相对恒定
- ✅ 硬件能精确测量电流
- ✅ 优先考虑能量效率

#### 7.2 选择速度控制的条件
- ✅ 需要稳定的速度输出
- ✅ 负载可能剧烈波动
- ✅ 允许百毫秒级延迟
- ✅ 对加速度平稳性有要求
- ✅ 需要长时间稳速运行

#### 7.3 混合控制策略（推荐）
结合两种方案的优点：
```
if (系统稳定 && IMU数据可信) {
    使用扭矩控制 + 轻量速度环稳定  // 快速反应
} else if (需要长期恒速 && 负载变化) {
    使用速度控制                    // 稳定补偿
} else {
    使用角度闭环 (级联控制)         // 精确定位
}
```

#### 1.3 对固件配置的影响

V4 原理图中母线电压采样采用 `R1=75k`、`R3=10k` 分压，比例约为 `(75k + 10k) / 10k = 8.5`，因此软件中的 V4 电压采样分支为：

```cpp
if (DengFOC) {
  Volts = analogReadMilliVolts(13) * 8.5 / 1000;
} else {
  Volts = analogReadMilliVolts(4) * 11 / 1000;
}
```

当实际硬件为 V4 板时，应将主程序中的板卡适配宏设置为：

```cpp
#define DengFOC 1   // 0: V3P板，1: V4板
```

否则程序会按 V3P 的 ADC 引脚和分压比例读取电压，可能导致 `CheckVinVolt()` 判断不准确，进而影响上电等待、欠压保护和电机标定流程。

## 07. `src/main.cpp` 主程序功能描述

`src/main.cpp` 是双轴云台固件的主入口文件，负责把供电检测、传感器初始化、电机标定参数管理、姿态解算和双电机力矩控制串联成完整的实时闭环。它运行在 ESP32 Arduino 框架下，依赖 DengFOC 驱动库、BMI160 IMU、EEPROM、低通滤波器和 PID 控制器。

### 1. 工程调用 Arduino 框架的方式

本工程并不是在业务代码中手动调用 Arduino 主入口，而是通过 PlatformIO 配置选择 Arduino 框架，由 Arduino-ESP32 框架在启动后自动调用 `setup()`，并持续循环调用 `loop()`。

PlatformIO 配置位于 `platformio.ini`：

```ini
[env:lolin32_lite]
platform = espressif32
board = lolin32_lite
framework = arduino
lib_deps = emotibit/EmotiBit BMI160@^0.3.3
```

其中 `framework = arduino` 表示编译和链接时使用 Arduino-ESP32 框架；`board = lolin32_lite` 表示目标开发板为 ESP32 LOLIN32 Lite。Arduino 框架内部已经提供底层 `main()`，工程只需要实现标准入口函数：

```cpp
void setup() {
  // 系统初始化
}

void loop() {
  // 周期性控制任务
}
```

其运行关系可以理解为：

```cpp
int main() {
  init();
  setup();

  while (true) {
    loop();
  }
}
```

实际的 `main()` 由 Arduino-ESP32 框架提供，工程代码不需要也不应该重复定义。`src/main.cpp` 通过 `#include <Arduino.h>` 引入 Arduino API，因此可以直接使用串口、延时、系统时间、ADC 和 I2C 等能力。

本工程中典型 Arduino API 使用如下：

| Arduino API | 在工程中的用途 |
|-------------|----------------|
| `Serial.begin(115200)` | 初始化串口调试输出 |
| `Serial.print()` / `Serial.println()` / `Serial.printf()` | 输出标定、错误和运行状态信息 |
| `delay()` | 上电等待、EEPROM 操作间隔、IMU 校准采样间隔 |
| `millis()` | 计算姿态积分时间间隔 |
| `analogReadMilliVolts()` | 读取母线电压 ADC 采样值 |
| `Wire` | 作为 BMI160 I2C 通信对象传入初始化函数 |

`lib/DengFOC` 下的驱动代码同样包含 `Arduino.h`，因此底层 FOC、AS5600 编码器、电流采样等模块也依赖 Arduino 框架提供的 GPIO、PWM、ADC、串口和 I2C 能力。

### 2. 全局配置与控制参数

主程序通过宏和全局对象定义系统运行条件：

| 配置/对象 | 作用 |
|----------|------|
| `#define DengFOC 0` | 选择适配的 DengFOC 驱动板版本，`0` 对应 V3P，`1` 对应 V4，主要影响电压采样引脚和分压系数 |
| `EEPROM_SIZE` | 设置 ESP32 EEPROM 模拟存储区大小为 32 字节 |
| `M0_Initialize_Align` / `M1_Initialize_Align` | 保存 M0/M1 零电角度的 EEPROM 地址 |
| `M0_Encoder_DIR` / `M1_Encoder_DIR` | 保存 M0/M1 编码器方向的 EEPROM 地址 |
| `UNDERVOLTAGE_THRES` | 欠压等待阈值，当前为 `11.1V` |
| `Calibration_Flag` | 标定模式开关，`1` 写入标定数据，`0` 读取已保存数据并进入运行 |
| `Y_loop` / `Z_loop` | 垂直轴和水平轴 PID 控制器 |
| `Y_Flt` / `Z_Flt` / `Z_Flt_Damping` | 力矩输出和水平轴阻尼用低通滤波器 |

其中 M0 被用作垂直方向控制电机，M1 被用作水平方向控制电机。程序中的 `Torget_0`、`Torget_1` 分别表示两路电机的目标力矩输出。

### 3. 上电初始化流程

`setup()` 完成系统启动阶段的全部准备工作：

1. 初始化串口，波特率为 `115200`，用于输出调试和标定信息。
2. 调用 `CheckVinVolt()` 检测电源电压，电压低于 `11.1V` 时持续等待，避免电机在欠压状态下开始标定或运行。
3. 调用 `DFOC_Vbus(12.6)` 设置驱动器供电电压参数。
4. 调用 `Bmi160Init()` 初始化 BMI160，设置 I2C 模式、陀螺仪量程和采样率。
5. 调用 `CalibrateGyro()` 对陀螺仪进行静态零偏校准。
6. 调用 `DFOC_enable()` 使能双电机驱动。
7. 初始化 EEPROM，并根据 `Calibration_Flag` 选择执行标定写入或快速读取流程。
8. 计算 M0 编码器角度和 IMU 姿态角之间的初始偏移 `now`，用于后续把 IMU 姿态映射到电机目标角度。
9. 调用一次 `GetAngle()` 更新姿态状态，随后进入主循环。

### 4. 电机标定与 EEPROM 参数保存

当 `Calibration_Flag = 1` 时，主程序进入标定写入模式：

1. 分别调用 `DFOC_M0_alignSensor(7)` 和 `DFOC_M1_alignSensor(7)` 执行两路电机传感器对齐，参数 `7` 为电机极对数。
2. 通过 `Get_M0_Dir()`、`Get_M1_Dir()` 获取编码器方向。
3. 将 M0/M1 零电角度和编码器方向写入 EEPROM。
4. 写入完成后停在 `while(1)`，提示用户把 `Calibration_Flag` 改回 `0` 并重新烧录，进入正常运行模式。

当 `Calibration_Flag = 0` 时，主程序进入快速启动模式：

1. 从 EEPROM 读取 M0/M1 零电角度和编码器方向。
2. 调用 `DFOC_M0_Set_alignSensor()`、`DFOC_M1_Set_alignSensor()` 恢复标定参数。
3. 不再重复执行电机对齐动作，从而缩短上电启动时间，并减少机械结构不必要的运动。

### 5. BMI160 姿态采集与融合

`GetAngle()` 是主程序的姿态更新函数，负责把 BMI160 原始数据转换成控制可用的姿态角：

1. 读取加速度计原始值 `ax/ay/az` 和陀螺仪原始值 `gx/gy/gz`。
2. 根据云台安装方向进行坐标轴映射，当前映射为 `bx = ay`、`by = -ax`、`bz = az`，陀螺仪映射为 `cx = gy`、`cy = -gx`、`cz = gz`。
3. 将加速度计数据换算为 g 单位，将陀螺仪数据换算为 dps 单位。
4. 用 `atan2()` 根据加速度计算 X/Y 倾角。
5. 对陀螺仪数据进行死区处理，过滤微小噪声，减少角度积分漂移。
6. 对陀螺仪角速度进行低通滤波。
7. X/Y 轴使用互补滤波融合陀螺仪积分角度和加速度角度，Z 轴主要使用陀螺仪积分得到偏航角。

该函数输出的 `last_angleX`、`last_angleY`、`last_angleZ` 是主循环控制的核心姿态输入。

### 6. 主循环双轴控制逻辑

`loop()` 是实时控制循环，持续完成 FOC 刷新、姿态更新、误差计算和力矩输出：

1. 调用 `runFOC()` 更新 DengFOC 底层闭环和 PWM 输出。
2. 调用 `GetAngle()` 获取最新姿态角。
3. 垂直轴 M0 使用 `last_angleY` 和初始偏移 `now` 计算目标位置映射值：

   ```cpp
   angle1_error_0 = M0_Dir * last_angleY - 0 + now;
   ```

4. 当垂直轴误差小于 `0.3` 度时，M0 输出力矩置零，避免小误差下反复抖动；否则通过 `Y_loop` PID 和 `Y_Flt` 低通滤波计算 `Torget_0`。
5. 水平轴 M1 使用 `last_angleZ` 作为偏航误差输入，通过 `Z_loop` PID 和 `Z_Flt` 低通滤波计算 `Torget_1`。
6. 如果水平轴误差连续多次不变化，程序将 `Z_loop.I` 临时置零并清除目标输出，避免静止时积分累积导致后续突跳。
7. 最后调用 `DFOC_M0_setTorque()` 和 `DFOC_M1_setTorque()` 输出两路电机力矩。

### 7. 水平轴前馈补偿与阻尼

M1 的最终输出不仅包含 PID 结果，还叠加了前馈补偿和角度阻尼：

```cpp
DFOC_M1_setTorque(
  M1_Dir * (Torget_1 + 2.9 * sin(last_angleY * PI / 180)) +
  (Z_Flt_Damping(DFOC_M1_Angle()) - DFOC_M1_Angle())
);
```

其中 `2.9 * sin(last_angleY * PI / 180)` 用于根据俯仰角补偿重力或结构姿态带来的负载变化；`Z_Flt_Damping(DFOC_M1_Angle()) - DFOC_M1_Angle()` 形成与角度变化相关的阻尼项，用于抑制水平轴震荡，使云台运动更平稳。

### 8. 辅助函数功能

| 函数 | 功能 |
|------|------|
| `GetVinVolt()` | 根据板卡版本读取对应 ADC 引脚并换算母线电压 |
| `CheckVinVolt()` | 上电阶段循环检测电压，低于阈值时等待 |
| `WriteFloatToEEPROM()` / `WriteIntToEEPROM()` | 将标定得到的浮点数或整型数据写入 EEPROM，并进行地址越界检查 |
| `ReadFloatToEEPROM()` / `ReadIntToEEPROM()` | 从 EEPROM 读取标定数据，并进行地址越界检查 |
| `Bmi160Init()` | 初始化 BMI160，设置 I2C 地址、陀螺仪量程和采样率 |
| `CalibrateGyro()` | 静止采样 1000 次陀螺仪数据，计算 X/Y/Z 轴零偏 |
| `GetAngle()` | 读取 IMU、完成坐标映射、滤波、角度融合和姿态更新 |

`src/main.cpp` 构成的实时闭环可以概括为：


## 10. 数据流详解

### 1. 传感器数据处理流

```
IMU原始数据 (int16_t)
    ↓
缩放转换 (× 0.061 → g单位, × 0.00763 → dps单位)
    ↓
陀螺仪死区过滤 (< 0.05 dps)
    ↓
角度积分 (使用时间间隔)
    ↓
IMU互补滤波 + 加速度计融合
    ↓
融合角度输出 (angleX, angleY)
    ↓
上层控制系统
```

### 2. 编码器数据处理流

```
AS5600 12位输出 (0~4095)
    ↓
转换为角度 (θ = RAW × 360/4096)
    ↓
圈数计数 (检测跨越0°/360°)
    ↓
计算完整角度 (full_rotations × 360 + θ)
    ↓
速度计算 (Δθ / Δt)
    ↓
低通滤波
    ↓
控制系统反馈
```

### 3. 控制输出流

```
电流目标值 Iq_target
    ↓
Park反变换 (dq → αβ)
    ↓
Clark反变换 (αβ → abc)
    ↓
SVPWM调制
    ↓
PWM占空比 (0~100%)
    ↓
GPIO输出 → 功率级驱动
```

## 12. 高级特性

### 1. 低通滤波

系统使用一阶低通滤波器(LowPassFilter类)对各环节数据进行平滑处理：

```cpp
class LowPassFilter {
    float Tf;      // 时间常数 (单位: 秒)
    float alpha;   // 滤波系数 = Tf / (Tf + Ts)
    float y_prev;  // 上一次输出
};

// 一阶低通滤波方程: y = α·x + (1-α)·y_prev
```

**应用场景**:
- 垂直电机速度: Tf=10ms
- 水平电机速度: Tf=10ms
- 电流反馈: Tf=5ms

### 2. 动态限幅

PID输出通过 `output_ramp` 参数限制变化率，防止突变：

```cpp
if (|output - output_prev| > ramp × Ts) {
    output = output_prev + sign(output - output_prev) × ramp × Ts;
}
```

### 3. 角度归一化

```cpp
float _normalizeAngle(float angle) {
    // 将角度归一化到 [-π, π] 范围
    while (angle > PI) angle -= 2*PI;
    while (angle < -PI) angle += 2*PI;
    return angle;
}
```

### 1. Firewater 协议概述

#### 1.1 协议特点

**Firewater** 是 VOFA+ 上位机软件支持的轻量级实时数据通信协议，专门为资源受限的嵌入式系统设计。

| 特点 | 描述 | 优势 |
|------|------|------|
| **轻量级** | 包头仅3字节 | 传输开销低 |
| **高效率** | 支持变长数据 | 适应多种数据类型 |
| **可靠性** | CRC校验 | 错误检测 |
| **实时性** | 低延迟 | 适合实时监测 |
| **易集成** | 简单协议格式 | 易于嵌入式实现 |

#### 1.2 Firewater 数据包结构

| 字段 | 帧头 | 数据长度 | 数据内容 | 检验和 |
|------|------|---------|---------|--------|
| **名称** | Frame Header | Data Length | Data Content | Checksum |
| **标识值** | 0xAA | 0-255 | 实际数据 | CRC8 |
| **字节数** | 1 byte | 1 byte | N bytes | 1 byte |
| **功能** | 同步标记 | 后续数据长度 | 有效载荷 | 错误检测 |
| **说明** | 标识数据包开始 | 表示数据段字节数 | 由多个变量组成 | 数据完整性校验 |

**数据包总长度**: 3 + N 字节（其中N为实际数据长度）

### 6. CRC8 校验实现

```cpp
// CRC8校验码计算
uint8_t calculateCRC8(const uint8_t* data, uint16_t length) {
    uint8_t crc = 0;
    
    for (uint16_t i = 0; i < length; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x80) {
                crc = (crc << 1) ^ 0x31;  // CRC多项式
            } else {
                crc = crc << 1;
            }
        }
    }
    
    return crc;
}
```

## 14. 故障诊断

### 1. 常见故障

| 故障现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 电机不转 | 对齐失败 | 检查编码器连接，重新运行对齐 |
| 抖动明显 | PID参数不合适 | 调整 P/I/D 系数 |
| 角速度异常 | 陀螺仪漂移 | 重新校准陀螺仪 |
| 欠压报警 | 电源不足 | 检查电源，确保 > 11.1V |
| 编码器数据异常 | I2C通信故障 | 检查IIC线路和从地址 |
| 找不到 COM 口 | CH340/CH340K 驱动、USB 线或端口选择异常 | 更换 USB 线，确认设备管理器端口，并在 PlatformIO/Arduino 中选择同一 COM 口 |
| 标定后仍无法快速启动 | `Calibration_Flag` 未改回 `0` 或 EEPROM 未写入成功 | 确认串口输出“数据写入EEPROM成功”，再改为 `Calibration_Flag = 0` 后重新烧录 |
| 烧录失败或电机异常动作 | 12V 总电源未断开，烧录时电机驱动仍处于供电状态 | 烧录前断开 12V 总电源，仅保留 USB Type-C 连接 |
| 上电后机构卡住 | 电机位置靠近机械限位或线束干涉 | 上电前将云台调整到可自由运动的位置，避开卡位点 |

### 2. 调试接口

通过串口命令进行实时调试：
```
函数: serialReceiveUserCommand()
函数: serial_motor_target()
```
